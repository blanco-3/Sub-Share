# Sub-Share 프로젝트 보고서

> 이 보고서는 프로젝트 진행 과정을 복기하기 위해 지속적으로 업데이트됩니다.

---

## 프로젝트 주제

**Sub-Share** — 신뢰 없는 관계에서의 구독 비용 공동 결제 도구

온라인에서 처음 만난 개발자 팀이 고가 AI 구독(Claude Team, ChatGPT Team, Cursor Business 등)을 공동 구매할 때 발생하는 신뢰 문제를 스마트 컨트랙트로 해결하는 프로덕트.

---

## 활동 목적

ShardLab Product Intern Challenge 과제 수행 (48시간 제한)

과제 요구사항:
- 비용을 나누는 상황에서 깨져 있는 구체적인 순간 발견
- 문제를 깊이 있게 정의
- AI 도구로 인터랙티브 프로토타입 제작
- 1페이지 설명서 작성

---

## 활동 목표

1. 기존 정산 도구(토스, 카카오페이, Splitwise 등)가 해결하지 못하는 "신뢰 부재" 상황의 페인포인트를 정의
2. Web3(스마트 컨트랙트, Account Abstraction, 스테이블코인)가 억지가 아니라 자연스럽게 필요한 해결 구조를 설계
3. ShardLab의 미션("블록체인으로 실생활 문제 해결")과 정렬되는 프로덕트 제시

---

## 사용 기술

| 기술 | 용도 |
|------|------|
| React | 프로토타입 UI |
| Claude Artifacts | 프로토타입 제작 환경 |
| Reown AppKit | 소셜 로그인 + Account Abstraction |
| Solidity (설계) | 볼트 스마트 컨트랙트 |

| USDC | 스테이블코인 결제 (환율 문제 해결) |
| Base L2 | 배포 네트워크 (저렴한 가스비) |
| Reclaim Protocol | Claude 결제 증빙 기반 zkTLS 환급 청구 |

---

## 기술 선택 이유

### Reown AppKit
- 소셜 로그인(Google, Discord, GitHub) + 지갑 연결을 하나의 SDK로 통합
- Account Abstraction으로 지갑 없는 사용자도 스마트 계정 자동 생성
- 개발자뿐 아니라 일반 사용자까지 허들을 낮춤

### 스마트 컨트랙트 볼트
- 서로 일면식 없는 팀원 간 "신뢰"를 "코드"로 대체
- 선입금 잠금으로 먹튀 구조적 방지
- 만장일치 승인 또는 Reclaim 증명으로 월별 환급 청구 가능

### USDC (스테이블코인)
- 달러 기반이라 환율 변동 없음
- 크로스보더 팀(한국+베트남+인도 등)에서 통일된 결제 단위
- 기존 송금 수수료 제거

### Reclaim + reimbursement vault
- 선결제자가 Claude를 실제로 결제한 뒤 환급을 청구하는 구조로 문제를 더 구체화
- 팀원이 먼저 자금을 잠가 두기 때문에 환급 리스크를 낮춤
- Claude 영수증을 zkTLS로 증명하면 투표 지연 없이 환급 가능

## 문제 상황 및 목표

### 페인포인트
온라인에서 처음 만난 개발자 4명이 해커톤/프로젝트를 위해 AI 도구 Team 플랜($30/월)을 공동 구매하려 한다. 한 명이 전액 결제해야 하고, 나머지가 돈을 보낼지 보장이 없으며, 중간에 빠져도 강제할 수단이 없다.

### 문제의 레이어
1. **신뢰 부재** — 일면식 없는 사이. 실명도 모를 수 있음.
2. **선금 리스크** — 한 명이 전액 결제. 나머지가 안 보내면 쫓아갈 방법 없음.
3. **먹튀** — 한 달 쓰고 사라짐. 남은 사람 부담 급증.
4. **계약 기간 불일치** — 구독은 월 단위, 프로젝트는 3개월. 중간 탈퇴 시 잔여 비용 문제.
5. **분쟁 해결 수단 없음** — 친구면 대화로 풀지만, 모르는 사이에서는 불가능.
6. **크로스보더** — 다국적 팀이면 기존 정산 앱 자체가 작동 안 함.

### 기존 해결방식과 한계
- **토스/카카오페이**: 1/N 균등 분할만 디폴트. 차등 정산은 수동. 신뢰 전제.
- **Splitwise/Tricount**: 커스텀 분할 가능하나 누군가가 수동 설정 필요. 강제력 없음.
- **직접 송금**: 디스코드에서 "보내줘" → 안 보내면 방법 없음.

## 구체적 해결과정

### Phase 1: 문제 탐색 (Part 1-2)
- 초기 방향: 술자리 N빵 문제 → 계산기 수준으로 깊이 부족
- 방향 전환: "신뢰 없는 관계에서의 비용 분담"으로 확장
- ShardLab 리서치: Web3 R&D 조직 → "블록체인으로 실생활 문제 해결" 미션 확인
- 최종 방향: 고가 AI 구독의 팀 공동결제 + 스마트 컨트랙트 볼트

### Phase 2: 기존 앱 분석
- 토스 더치페이: 1/N 디폴트, 수동 차등 가능하나 신뢰 전제
- 카카오페이 정산: 직접 입력으로 차등 가능하나 강제력 없음
- Splitwise: 다양한 분할 방식 지원하나 결제 강제 수단 없음
- **공통 한계**: 모든 앱이 "신뢰"를 전제. 안 내도 강제할 수단 없음.

### Phase 3: 프로토타입 개발
- v1: 술자리 정산 계산기 (폐기 — 깊이 부족)
- v2: 볼트 기본 흐름 (생성 → 입금 → 분배)
- v3: 소셜 로그인 + Reclaim proof + 전체 라이프사이클

### Phase 4: 핵심 설계 결정
1. **선결제 환급 구조 구체화**: 관리자 자율 인출이 아니라 승인 또는 증명 기반 환급 구조로 변경
2. **소셜 로그인**: EOA 필수 → Reown AppKit으로 Google/Discord/GitHub 로그인 지원
3. **한/영 지원**: 글로벌 팀 사용 고려

## 결과(성과)

### 프로토타입
- 전체 라이프사이클을 인터랙티브하게 시뮬레이션하는 React 프로토타입 완성
- 홈 → 소셜 로그인 → 온보딩 → 볼트 생성 → 초대 → 입금 → 활성화(환급 청구) → 승인/zkTLS 청구 → 완료
- 한국어/영어 전환 지원

### 기술적 검증
- 스마트 컨트랙트 볼트 로직: 전부 기존 검증된 DeFi 패턴 (에스크로, 멀티시그, 타임락)
- Account Abstraction: Reown AppKit으로 구현 가능 확인
- Reclaim: Claude 영수증 증빙을 통한 환급 가속 경로 통합

## 배운점

1. **문제의 깊이가 해결의 깊이를 결정한다** — 처음 술자리 N빵으로 시작했을 때 계산기 수준의 해결밖에 안 나왔음. 문제 자체가 복잡하고 여러 레이어가 있어야 의미 있는 프로덕트가 나옴.

2. **Web3는 억지로 넣는 게 아니라, 필요한 곳에 자연스럽게 들어가야 한다** — 친한 사이의 정산에는 Web3가 필요 없음. "신뢰가 없는 관계"에서 "코드로 신뢰를 대체"하는 것이 Web3의 본질적 가치.

3. **사용자 관점 vs 빌더 관점** — 멀티시그, 에스크로, AA 같은 기술 용어보다 "선결제 후 환급을 독촉하지 않아도 된다"가 사용자 관점. 기술은 뒤에서 돌아가는 것.

4. **ShardLab 미션과의 정렬** — "블록체인으로 실생활 문제 해결"이라는 ShardLab의 미션을 이해하고, 그에 맞는 문제를 선택한 것이 방향 설정에 결정적이었음.

5. **범위 조절의 중요성** — 48시간 안에 할 수 있는 것과 없는 것을 구분하고, 핵심에 집중하되 확장 가능성은 설명서에서 논하는 전략.
